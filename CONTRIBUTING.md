# 贡献指南

欢迎来到 FreeODwiki！本项目致力于以科学、客观、无道德偏见的方式，整理和分享药物相关知识与体验报告。我们鼓励所有对药物话题有兴趣的朋友参与贡献，无论是修正错别字、补充内容，还是分享个人经验。

有关AI使用：不禁止AI的作品，只要符合事实，符合正常人的脑回路就可以接受。在[FOW规划与规范](文档/关于本站/FOW规划及规范.md)中也有AI使用的推荐方针。

## 贡献方式

- **Issues**：反馈问题、提出建议、报告错误。
- **Discussions**：讨论条目内容、项目发展、社区事务。
- **Pull Requests**：提交内容修正、新条目、体验报告等。

## 内容类型

- 条目修正：错字、语法、逻辑、格式统一、链接修复。
- 新条目：药物介绍、理论、应对措施、社会学分析等。
- 体验报告：仅限本人提交、修改和追加，他人不得编辑。

## 结构与格式要求

- 保持文件夹和文件的层次结构清晰。
- 同类条目（如药物介绍）请遵循统一格式。
- 修正条目时，除非是明显错误，建议用注释保留原内容，便于追溯。
- 体验报告请遵守[药物报告投稿指南](./文档/关于本站/药物报告投稿指南.md)。
- 遵守[FOW规划及规范](文档\关于本站\FOW规划及规范.md)中的要求。

## 内容规范

- 逻辑通顺、语法正确、符合常识。
- 不带极端道德偏见，不歧视任何群体。
- 尽量注明信息来源（科学文献、Wikipedia、网站、个人经验等）。
- 不提供医疗建议，不鼓励违法或危险行为。

## 技术注意事项

- 所有文档需适配 Wiki.js 部署，主页为 `home.md`，请勿误用 `README.md`。
- 不涉及命令行操作，手机也可贡献。

### 复制一般文本到Markdown的注意事项

+ 请注意文本中的某些元素是否意外构成Markdown语法，导致显示错误的问题。  
  例如：  
  `我昨天只睡了4个小时[前天更是只睡了3个小时](唉，感觉要猝死了)`
  会渲染为：  
  > 我昨天只睡了4个小时[前天更是只睡了3个小时](唉，感觉要猝死了)
  >
  这是错误的，正确的写法是使用转义字符`\`：  
  `我昨天只睡了4个小时\[前天更是只睡了3个小时\](唉，感觉要猝死了)`  
  会渲染为：  
  > 我昨天只睡了4个小时\[前天更是只睡了3个小时\](唉，感觉要猝死了)
  >

+ 请注意在Markdown中换行不会直接换行。  
  例如：  
  ```plain_text
  昨天，他只睡了4个小时
  前天更是只睡了3个小时
  意外猝死正在悄然酝酿
  ```
  会渲染为：  
  > 昨天，他只睡了4个小时
  前天更是只睡了3个小时
  意外猝死正在悄然酝酿
  >
  这是错误的，正确的写法是(请注意在结尾添加的两个空格)：  
  ```plain_text
  昨天，他只睡了4个小时  
  前天更是只睡了3个小时  
  意外猝死正在悄然酝酿  
  ```
  会渲染为：  
  > 昨天，他只睡了4个小时  
  前天更是只睡了3个小时  
  意外猝死正在悄然酝酿  
  >
  （上一个版本中这里写的是使用`<br>`，这里简要说明一下为什么更改。第一，很明显两个空格看上去很明显要简洁不少也少打一点字。第二，`<br>`并不是一个标准的Markdown语法他是Markdown内嵌HTML，因此并不是所有Markdown解释器都支持这一点，也就是不好的编程习惯了，本PR的提交者在自己的网站中就使用的是两个空格，不过我相信可能有些人会觉得显式优于~~隐形~~隐式因此保留了这一段。）

这些错误可以通过预览文件避免，等到你做过这些事情，知道要避免踩坑的注意事项了也就不会犯这些错误了，加油哦！

## 参与流程

提交文档很简单，不需要电脑，手机就行。不涉及任何命令行操作，这里给一个简单的视频教程，[点击这里查看](https://x.com/SalviaSWC/status/2001249359095386327)

用文字来说就是：
1. Fork 本仓库，或直接在 Github 网页端编辑。
2. 按上述规范修改或新增内容。
3. 提交 Pull Request，简要说明更改内容。
4. 等待审核与合并。

## 其他说明

- 本站不提供任何交易，不鼓励或教唆药物滥用。
- 所有贡献者请保持理性、尊重他人。
- 如发现内容有误或不当，请及时反馈。
- 使用受版权保护的材料时，一定要获得持有人的允许。

## 药物报告投稿指南

你可以直接联系推特上FreeODwiki的人员提供报告，但最好使用你自己的账号在Github上提交。

投稿药物报告和编辑条目的流程是一样的，区别是你需要在`./报告`目录下创建含有你的药物报告的文档。暂定文档标题格式如下`自拟的药物报告标题 药物名称(联用时以空格分割).md`。


感谢你的参与，让 FreeODwiki 更加完善！
