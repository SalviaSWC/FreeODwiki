## 内容规范

- 逻辑通顺、语法正确、符合常识。
- 不带极端道德偏见，不歧视任何群体。
- 尽量注明信息来源（科学文献、Wikipedia、网站、个人经验等）。

### 复制一般文本到Markdown的注意事项

+ 请注意文本中的某些元素是否意外构成Markdown语法，导致显示错误的问题。例如：  
  ```markdown
  我昨天只睡了4个小时[前天更是只睡了3个小时](唉，感觉要猝死了)
  ``` 
  会渲染为：  

  ---
  我昨天只睡了4个小时[前天更是只睡了3个小时](唉，感觉要猝死了)
  
  ---
  这是错误的，正确的写法是使用转义字符`\`：  
  ```markdown
  我昨天只睡了4个小时\[前天更是只睡了3个小时\](唉，感觉要猝死了)
  ```  
  会渲染为：
  
  ---
  我昨天只睡了4个小时\[前天更是只睡了3个小时\](唉，感觉要猝死了)

  ---

+ 请注意在Markdown中换行不会直接换行。例如：  
  ```markdown
  昨天，他只睡了4个小时
  前天更是只睡了3个小时
  意外猝死正在悄然酝酿
  ```
  会渲染为：
  
  ---
  昨天，他只睡了4个小时  前天更是只睡了3个小时  意外猝死正在悄然酝酿
  
  ---
  这是错误的，正确的写法是（请注意在结尾添加的两个空格）：  
  ```markdown
  昨天，他只睡了4个小时  
  前天更是只睡了3个小时  
  意外猝死正在悄然酝酿  
  ```
  会渲染为：
  
  ---
  昨天，他只睡了4个小时  
  前天更是只睡了3个小时  
  意外猝死正在悄然酝酿
  
  ---
  
  > 上一个版本中这里写的是使用`<br>`，这里简要说明一下为什么更改。第一，很明显两个空格看上去很明显要简洁不少也少打一点字。第二，`<br>`并不是一个标准的Markdown语法他是Markdown内嵌HTML，因此并不是所有Markdown解释器都支持这一点，也就是不好的编程习惯了，本PR的提交者在自己的网站中就使用的是两个空格，不过我相信可能有些人会觉得显式优于~~隐形~~隐式因此保留了这一段。
{.is-info}

  要分段请使用Markdown的方式——添加一个空行

这些错误可以通过预览文件避免，等到你做过这些事情，知道要避免踩坑的注意事项了也就不会犯这些错误了，加油哦！

## 参与流程

提交文档很简单，不需要电脑，手机就行。不涉及任何命令行操作，这里给一个简单的视频教程，[点击这里查看](https://x.com/SalviaSWC/status/2001249359095386327)

用文字来说就是：
1. Fork 本仓库，或直接在 Github 网页端编辑。
2. 按上述规范修改或新增内容。
3. 提交 Pull Request，简要说明更改内容。
4. 等待审核与合并。

## 药物报告投稿指南

你可以直接联系推特上FreeODwiki的人员提供报告，但最好使用你自己的账号在Github上提交。你也可以在现存的freeodwiki渲染站点上注册账号编辑。

投稿药物报告和编辑条目的流程是一样的，区别是你需要在`./报告`目录下创建含有你的药物报告的文档。暂定文档标题格式如下`自拟的药物报告标题 药物名称(联用时以空格分割).md`。

## 一般文档规划

对于每一类的文档，都需要用一个文件夹专门存储此类文档文件。除非真的有必要且不会引起歧义，否则不要在该文件夹里面再建文件夹，以减少无意义的嵌套。

在这个文件夹中，通常需要有一个文件作为这个文件夹的索引页或主页的角色。这个文件应该被统一命名为`home.md`。

### 文件夹的使用规范

若无必要，请勿增加文件夹的嵌套。虽然它看上去可能很好看或有条理，实际上却增加了路径计算的开支，并且会导致用于处理文档的LLM难以理解文件夹之间的嵌套关系，间接增加编写成本。

## 文档维护指南

#### 目录索引

目录是文档内的索引，具有告诉读者本文档有何内容的作用。如果目录缺失或者不对，请及时修正

#### 超链接

超链接让文档内部与文档外部形成一个整体，让整个站点变得易于浏览。如果超链接有不对的地方，务必及时修正；如果一些本应有超链接的部分没有，请补上

#### 文件内容

如果你发现文档的一个指向`/文件/`目录的文件缺失，请将相应的文件补上。

## 翻译文档

翻译的文档请注明来源。以下为推荐的流程：

*例子是psywiki，因为它的CC协议与我们的兼容，无侵权隐患*。其他的网站的翻译也可以类似处理。

#### 获取html源代码

通过电脑浏览器获取网站的html源代码，将其复制，作为翻译文档的起始物料。

#### 获取markdown源代码粗品

打开[脚本](/关于本站/脚本/转化html到markdown并删除冗余内容.py)，复制到本地并运行。你会看到一个粗糙的图形界面，粘贴到输入框中，并点击按钮，获得markdown粗品，是英文的。*这个脚本目前是专门适配psywiki的，所谓专门适配只是对于psywiki的某些部分做了一个特判处理，优化输出的文件罢了*

#### AI法精制Markdown源代码

<mark>请贡献者和阅读者注意，如果你发现了一个经常被错译的名词或已经存在但你认为翻译不当的名词，请随时提出Issue或直接Pull Request更新这里的术语表</mark>

    - 在输出的开头，文件标题的后的第一段，需要给出一个"返回"链接，如下:`[◀返回](./home.md)`
