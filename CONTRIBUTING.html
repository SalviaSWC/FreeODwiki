    复制一般文本到Markdown的注意事项

    • 请注意文本中的某些元素是否意外构成Markdown语法，导致显示错误的问题。例如：
      markdown 我昨天只睡了4个小时[前天更是只睡了3个小时](唉，感觉要猝死了) 会渲染为：

    我昨天只睡了4个小时前天更是只睡了3个小时

    这是错误的，正确的写法是使用转义字符\：
    markdown 我昨天只睡了4个小时\[前天更是只睡了3个小时\](唉，感觉要猝死了)
    会渲染为：

    我昨天只睡了4个小时[前天更是只睡了3个小时](唉，感觉要猝死了)

    • 请注意在Markdown中换行不会直接换行。例如：
      markdown 昨天，他只睡了4个小时 前天更是只睡了3个小时 意外猝死正在悄然酝酿 会渲染为：

    昨天，他只睡了4个小时 前天更是只睡了3个小时 意外猝死正在悄然酝酿

    这是错误的，正确的写法是（请注意在结尾添加的两个空格）：
    markdown 昨天，他只睡了4个小时 前天更是只睡了3个小时 意外猝死正在悄然酝酿 会渲染为：

    昨天，他只睡了4个小时
    前天更是只睡了3个小时
    意外猝死正在悄然酝酿

    上一个版本中这里写的是使用<br>，这里简要说明一下为什么更改。第一，很明显两个空格看上去很明显要简洁不少也少打一点字。第二，<br>并不是一个标准的Markdown语法他是Markdown内嵌HTML，因此并不是所有Markdown解释器都支持这一点，也就是不好的编程习惯了，本PR的提交者在自己的网站中就使用的是两个空格，不过我相信可能有些人会觉得显式优于~~隐形~~隐式因此保留了这一段。 {.is-info}

    要分段请使用Markdown的方式——添加一个空行

    这些错误可以通过预览文件避免，等到你做过这些事情，知道要避免踩坑的注意事项了也就不会犯这些错误了，加油哦！

    一般文档规划

    对于每一类的文档，都需要用一个文件夹专门存储此类文档文件。除非真的有必要且不会引起歧义，否则不要在该文件夹里面再建文件夹，以减少无意义的嵌套。

    在这个文件夹中，通常需要有一个文件作为这个文件夹的索引页或主页的角色。这个文件应该被统一命名为home.md。

    文件夹的使用规范

    若无必要，请勿增加文件夹的嵌套。虽然它看上去可能很好看或有条理，实际上却增加了路径计算的开支，并且会导致用于处理文档的LLM难以理解文件夹之间的嵌套关系，间接增加编写成本。

    文档维护指南

    目录索引

    目录是文档内的索引，具有告诉读者本文档有何内容的作用。如果目录缺失或者不对，请及时修正

    超链接

    超链接让文档内部与文档外部形成一个整体，让整个站点变得易于浏览。如果超链接有不对的地方，务必及时修正；如果一些本应有超链接的部分没有，请补上

    文件内容

    如果你发现文档的一个指向/文件/目录的文件缺失，请将相应的文件补上。
